Ubuntu Linux 与 Windows 7双系统安装教程（图文）

Posted on 2015-04-20 Views: 99+

Ubuntu Linux 与 Windows 7双系统安装教程

前期准备

1、 备份你的重要数据到其他设备上，以防不测
2、 准备linux镜像。可以到ubuntu官网下载iso格式的文件。（注意：如果你是双显卡，不要随便下。amd的双显卡请选择含有“amd”字样的iso镜像，如果是N卡，额，三思。否则会出现错误）Get Ubuntu
3、 制作系统盘。包括一个u盘和制作系统盘的工具，大白菜和其他的都行。这里推荐UltraISO，怎么制作系统盘请上网搜素。
4、 Easybcd软件。可以到官网下，修改引导用的。
5、 如果有可能，再制作一个带有pe系统的Windows 系统盘，以防不测。（网上带毒的很多，注意杀毒）

如果在安装过程中遇到问题，请参考“FAQs”

点击图片可以查看大图

第一步 给linux 系统分配空间

1、右击“我的电脑”，选择“管理”

2、选择“磁盘管理”

3、选择一个比较大的磁盘割一点空间出来。在该盘符上右击，选择“压缩卷”。Linux一般不需要太大空间，这里我选择20G，确定。

第二步 更改bios启动顺序

1、重启，进入bios。（不同的电脑进入bios的方式不同，具体请上网搜索）

2、设置u盘为第一启动方式，我用的是光盘安装，所以选择的是CD-ROM，如果是用u盘安装，则选择USB，电脑没查U盘，所以没显示。（注意：不同主板修改的地方不同，具体请上网搜索）

第三步 准备安装，创建分区

1、启动电脑，如果设置成功，会进入ubuntu的欢迎界面。选择“安装”。

2、“安装类型”选择“其他选项”，否则可能会把Windows7 覆盖！！！

3、继续，找到我们刚刚分配给ubuntu的那个盘符，我这里是 /dev/sda2 ntfs ，看盘符大小就行。

4、选择“空闲”

5、首先，创建引导分区，分区格式“ext4”，挂载点选择“/boot”，大小200M就可以了。点下面的“+”号

6、接着，创建根目录分区“/”，分区格式选择“ext4”，挂载点选择“/”，大小不少于前面要求的6.5G，我这里就选择10G了，类似于Windows下系统盘。

7、然后，创建交换分区，大小与电脑的内存大小相同就行，我选择的是4G，与虚拟内存是一个原理。

8、最后，创建“/home”分区，分区格式ext4，挂载点“/home” ，把剩下的所有空间全部分给它，这个分区相当于Windows下除了系统盘之外的所有盘。

第四步 选择启动引导器

注意：接下来一步很重要，如果弄错，无法进入系统，包括Windows！！！

1、安装启动引导器的设备，在下拉菜单中找到“/boot”的分区，如果没有创建“/boot”，选择“/”，我这里是“/dev/sda5”，这样就是Windows引导ubuntu。如果不选，默认是ubuntu引导Windows，我第一次就是这样，但是发现没法进Windows，还是建议用Windows引导，毕竟我们还是更熟悉Windows。

就快完成了

第五步 设置用户名和密码，等待安装

1、继续，设置用户名和密码，linux系统是必须设置密码的。

2、点击继续，就看到安装界面了，耐心的等待吧（如果你处于联网状态，会下载一些东西，可以选择跳过）。

小提示：安装ubuntu的时候可以会出来介绍，点击其中的链接，你会神奇的发现，安装系统的时候竟然还能看网页！Windows可没这么好的用户体验。

一段时间的等待之后，提示“安装完成，请重新启动”。

还没完哦。

第六步 在Windows 7下添加进入Ubuntu 系统的条目

1、拔出U盘，或者进入bios将硬盘的启动顺序调到第一个，一般是“SATA”

2、重新启动，进入Windows，

3、打开，easybcd，选择“添加新条目”

4、操作系统选择“Linux/BSD”，类型选择“GRUB/Legacy”，名称任意，驱动器选择ubuntu的/boot分区，看大小就行，我这里是分区-2 （Linux -190MiB）。如果前面操作时没有分配“/boot”分区，选择“/”分区，与前面设置的“启动引导器”相同。,

5、“添加条目”，保存设置，退出，操作完成。

第七步 快尝试下Ubuntu吧

1、 现在，重启电脑，会提示Windows 7 和 Ubuntu 的选择项，选择Ubuntu，就可以进入令人激动的Ubuntu 系统啦。

FAQs

出现“分区无法对齐”的警告怎么办？

答：这一般是格式不对。正常情况下在Windows系统中直接压缩出空间就行，但是有时会出这种“分区对齐”问题。上次同学想在希捷的移动硬盘中装Ubuntu系统，由于希捷硬盘的格式似乎与标准不一致，会出现无法对齐问题。
解决方法：在Windows 下弄出一段对齐的空间，即在空闲空间上新建卷之后再删除，注意左右不要有其他空闲空间，以免合并之后再次出现问题）。进入Ubuntu系统创建分区时，点击下面的“change”或者“-”，把先前分配出来的空间变成空闲，然后其他操作同正常流程。

关于创建分区，必须都创建吗？

答：否，除了根分区“/”，其他分区都是可选的，建议都有。
Linux下的分区跟Windows下的分区类似。
/ ： 根目录，与Windows下的系统盘类似
/boot ： 引导分区，Windows下也有类似的，但是不用关心。
Swap ： 交换分区，虚拟内存
/home： 用户分区，类似Windows下其他盘

安装完，进入Ubuntu系统时出现图形界面错误之类的警告，或者图形界面显示不正常？

答：你是双显卡吧，请注意看前期准备里的提示。Linux主打的不是个人PC，所以图形界面上会有些缺陷，默认是没有独显的驱动的。
解决方案：如果你是amd的独显，选择含有“amd”字样的镜像文件，重新安装一遍即可解决。如果你是“NVIDIA”显卡或者没有带“amd”显卡驱动的镜像，解决会麻烦些，我没有N卡，所以没有试过。请上网搜索“Linux 双显卡”，装上相关的驱动就可以。

Windows 7 系统进不去了，怎么办，文件会不会丢失？

答：第四步 选择启动引导器，没注意看吧，MBR被破坏了。别着急，容易解决。
解决方案：用一个带有pe的Windows 系统盘，进入win pe系统，进入方式跟Ubuntu相同，也是改bios，在pe里选择Diskgenius，选择“硬盘”->“重建主引导记录（MBR）”，重启电脑就行。

玩腻了，我想删除？

解决方案：
直接在Windows 下删除分配给Ubuntu系统的空间即可，别忘了用easybcd删除ubuntu的条目。

安装好麻烦，还有其他方案吗？

答：有，wubi安装，或者装在虚拟机里。
虚拟机安装比较简单，不会的上网搜一下就行。
至于wubi安装，就像在Windows上装一个程序，我没试过，网上说的缺点有：
1、 一休眠或挂起就会死机，不能从休眠或挂起状态恢复，只能强行关机然后重启。因为休眠和待机需要使用FUSE和基于文件的交换。建议用Ubuntu Tweak禁用休眠和挂起功能。
2、 如果安装分区以外的分区也有名为“Ubuntu”的文件夹的话，卸载时可能会把其它分区的Ubuntu文件夹误删除，所以建议不要在安装分区以外的分区自行建立名为“Ubuntu”的文件夹。
3、 删除Windows分区里的文件时不能使用回收站，会被直接删除。
4、 用Live CD启动不能访问虚拟分区里的文件。一旦系统崩溃，文件可能无法取回！建议把所有个人文件都存放到Windows分区中。
5、 进入在Windows里安装的Ubuntu时，如果突然断电的话，系统很容易崩溃，因为这样安装的Ubuntu系统会被当作是Windows系统下的一个文件，断电时这个文件会受损，启动或运行就会不正常了。

其它问题？

答：不要客气，快来打扰我吧，留言或者发邮件都行。
